# 🤖 Machine Learning Projects

Una raccolta di progetti personali di **Machine Learning supervisionato e non supervisionato**, sviluppati interamente a mano — senza l'ausilio di AI generativa per la scrittura del codice.

---

## 🎯 Obiettivo

L'obiettivo di questa repository è acquisire familiarità pratica con le librerie più utilizzate nell'ecosistema Machine Learning, costruendo progetti reali dall'esplorazione dei dati fino alla valutazione del modello.

Ogni progetto è un'occasione per:

- Applicare tecniche di preprocessing, feature engineering e model selection in modo consapevole
- Comprendere a fondo il funzionamento delle API delle principali librerie ML
- Sviluppare un approccio metodico al ciclo di vita di un progetto data-driven
- Consolidare le best practice per evitare errori comuni (data leakage, overfitting, ecc.)

---

## 🛠️ Stack tecnologico

Le librerie e i framework utilizzati nei progetti includono, tra gli altri:

| Libreria | Utilizzo principale |
|---|---|
| `scikit-learn` | Modelli supervisionati/non supervisionati, pipeline, preprocessing |
| `pandas` | Manipolazione e analisi dei dati |
| `numpy` | Operazioni numeriche e vettorializzazione |
| `matplotlib` / `seaborn` | Visualizzazione dei dati e dei risultati |
| `xgboost` / `lightgbm` | Gradient boosting |
| `pytorch` | Reti neurali e deep learning |

## 📁 Struttura della repository

```
ml-projects/
│
├── supervised/
│   ├── project-name/
│   │   ├── data/
│   │   ├── notebooks/
│   │   ├── src/
│   │   └── README.md
│   └── ...
│
├── unsupervised/
│   ├── project-name/
│   │   ├── data/
│   │   ├── notebooks/
│   │   ├── src/
│   │   └── README.md
│   └── ...
│
└── README.md
```

Ogni progetto ha una propria cartella con:
- `data/` — dataset utilizzato (o istruzioni per scaricarlo)
- `notebooks/` — notebook Jupyter con l'analisi esplorativa e il codice
- `src/` — eventuale codice modulare e riutilizzabile
- `README.md` — descrizione del progetto, dataset, approccio e risultati
